What it does

Pallas embeds itself in Claude Code as a senior athenahealth integration engineer. It does not just answer questions — it proactively protects you from the data-loss patterns and compliance mistakes that even experienced developers make against the athenaOne, DataView, and FHIR R4 surface.

Proactive safety

Flags PATIENTID = CHARTID joins (46% silent data loss), missing DELETEDDATETIME filters, missing CONTEXTID for service accounts, hardcoded credentials, and API code without rate-limit retry.

Teaches the "why"

Doesn't just say "add this filter." Explains the clinical, regulatory, or architectural reason behind every safety rule, so the lesson sticks across the next ten projects.

Curated knowledge base

828 DataView views with descriptions, 16K+ columns, 1.3K foreign-key relationships, 1.9K API/FHIR/workflow docs, plus identity patterns and gotchas accumulated from real integration work.

Learning loop

Every interaction feeds back. Low-risk patterns auto-merge into the KB; high-risk schema discoveries queue for human review. The tool gets smarter from every developer's hard-won lessons.

Slash commands

Type any of these in Claude Code to launch a guided workflow.

CommandDescription
/onboardGuided onboarding for new athenahealth developers (DataView, API, FHIR, or Marketplace tracks)
/sqlGenerate safe DataView SQL — CONTEXTID, soft-delete, and verified joins baked in
/athena-apiGenerate API integration code — OAuth2, rate-limit retry, structured error handling
/diagnoseRoot-cause analysis of an athena error with fix instructions and prevention guidance
/review-athenaScan a project for anti-patterns and safety issues
/validatePre-deployment safety check with critical/warning/info report
/explainDeep-dive on any athena concept — CONTEXTID, PATIENT vs CHART, soft-delete, etc.
/workflowEnd-to-end clinical/admin workflow guidance — scheduling, claims, lab results, etc.
/review-candidatesMaintainers: approve or reject KB update candidates from the learning loop

Setup

  1. Install via Claude Code marketplace

    From inside Claude Code:

    /plugin install athena-tools@claude-community
  2. Or install via npm (user-level config)

    From any terminal:

    npx pallas-athena-tools setup

    Installs MCP server config, 9 slash commands, and the safety persona into ~/.claude/. Works in every project.

  3. Verify

    Open Claude Code and ask:

    Use athena_explain_view on the PATIENT view

    You should see all 113 columns, the PII flags, the soft-delete requirement, and the PATIENTID/CHARTID identity warning.
